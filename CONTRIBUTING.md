# Contributing to Opspace

Opspace is in early development. The goal is to build a focused personal operating system, not a generic productivity tool.

## Principles

Before adding anything, check:

1. Does this improve capture speed?
2. Does this help execution?
3. Does this support review and reflection?
4. Does this avoid becoming a Notion clone?

## Development Rules

- Keep features minimal and opinionated
- Avoid unnecessary abstraction
- Prefer simple solutions
- Optimize for long-term maintainability

## Workflow

- Create a branch: feature/<name>
- Make small, focused changes
- Use clear commit messages

## Commit Convention

- feat: new feature
- fix: bug fix
- docs: documentation
- refactor: code improvement
- chore: setup or maintenance

## Scope Control

Do NOT add:

- template systems
- drag-and-drop builders
- generic database features
- over-configurable UI
